# Travel Planning Example

This example reimplements the AutoGen [AgentChat travel-planning scenario](https://github.com/microsoft/autogen/blob/027ecf0a379bcc1d09956d46d12d44a3ad9cee14/python/docs/src/user-guide/agentchat-user-guide/examples/travel-planning.ipynb#L4) as a DMAS-Forge application. It preserves the original four-agent setup and round-robin conversation flow, where execution stops when a message contains `TERMINATE`.

## Architecture

The application has five workflow services deployed in separate containers:

- **TravelPlannerAgent** (`workflow/planner_agent.go`) -- creates a travel itinerary draft based on the user request.
- **LocalAgent** (`workflow/local_agent.go`) -- adds local activities and place recommendations.
- **LanguageAgent** (`workflow/language_agent.go`) -- reviews the current plan and adds language and communication guidance.
- **TravelSummaryAgent** (`workflow/travel_summary_agent.go`) -- integrates all suggestions into a complete final plan and emits `TERMINATE` when done.
- **TravelCoordinator** (`workflow/travel_coordinator.go`) -- orchestrates a strict round-robin loop: `planner_agent -> local_agent -> language_agent -> travel_summary_agent -> ...` and checks for `TERMINATE` after every message.

Like the original AutoGen `RoundRobinGroupChat`, all agents see shared conversation context. The coordinator renders the transcript and passes it to each next speaker.

## Differences from the AutoGen Notebook

- The original notebook runs in-process in Python using `RoundRobinGroupChat`; this version is deployed as distributed services with DMAS-Forge.
- `TextMentionTermination("TERMINATE")` behavior is preserved as a case-sensitive substring check on each produced message.
- Instead of console streaming output, this example returns a structured response containing the full message trace, stop reason, and final plan.
- We added a statement to the TravelSummaryAgent system prompt, instructing it to not ask questions and make reasonable defaults instead. This avoids semi-infinite loops with models like GPT-5.4-nano who keep asking for questions without a human-in-the-loop.

## Setup

Edit `wiring/example_model.json` with your API key, model name, and URL:

```json
{
    "name": "gpt-5.4-nano",
    "url": "https://api.openai.com/v1",
    "key": "your-api-key-here"
}
```

## Build and Run

```bash
cd examples/travel-planning/wiring
go run main.go -w docker -o build -modfile=./example_model.json
cd build/docker
cp ../.local.env .env
docker compose build && docker compose up -d
```

## Usage

```bash
curl -sG 'http://localhost:12345/Plan' \
  --data-urlencode 'task=Plan a 3 day trip to Nepal.' | jq
```

Expected high-level response shape:

```json
{
  "Ret0": {
    "Messages": [
      {"Source": "user", "Content": "Plan a 3 day trip to Nepal."},
      {"Source": "planner_agent", "Content": "..."},
      {"Source": "local_agent", "Content": "..."},
      {"Source": "language_agent", "Content": "..."},
      {"Source": "travel_summary_agent", "Content": "... TERMINATE"}
    ],
    "StopReason": "Text 'TERMINATE' mentioned",
    "Terminated": true,
    "FinalPlan": "..."
  }
}
```
